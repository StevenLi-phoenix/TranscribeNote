# Notetaker Enhancement Plan

> Making Notetaker the most attractive macOS transcription app

## Round 1 — 2026-03-24

### AI-Powered Intelligence

- [ ] **Chat with Transcript** -- Add a conversational AI panel in SessionDetailView where users can ask questions about a session's transcript ("What did we decide about the deadline?", "Summarize the budget discussion"). Uses the already-configured LLM engine with transcript context injected as system prompt. This is table-stakes in 2026 -- Otter, Granola, Fireflies, and Jamie all offer it. The local-first angle means conversations never leave the machine.

- [ ] **Auto-Generated Action Items** -- After recording ends, automatically extract action items (tasks, decisions, follow-ups) from the transcript as a separate structured list with checkboxes. Store as a new `ActionItem` SwiftData model linked to the session. Users can mark items complete, assign due dates, and copy them as a task list. Fireflies and Fathom built their businesses on this.

- [ ] **Smart Auto-Tagging** -- Use the LLM to automatically generate 3-5 semantic tags for each session upon completion (e.g., "product-review", "1-on-1", "brainstorm", "hiring"). Store in the existing `RecordingSession.tags` array. Display as colored pills in SessionListView. Enable tag-based filtering in the sidebar. Transforms the session list from a chronological dump into an organized knowledge base.

- [ ] **AI Recipes (Custom Prompt Templates)** -- Inspired by Granola's Recipes: let users save reusable prompt templates for different meeting types (e.g., "Sprint Retro" extracts what went well / what didn't / action items; "Sales Call" extracts objections, next steps, pricing discussed; "Lecture Notes" produces study-guide format). Store as JSON in Application Support. Apply via a dropdown in the summary toolbar. This turns the LLM from a generic summarizer into a domain-specific assistant.

- [ ] **Cross-Session Knowledge Search** -- Add a global "Ask your notes" search bar (Cmd+Shift+F) that queries the LLM with context from ALL sessions, not just one. Uses RAG-style approach: full-text search narrows to relevant transcript chunks, then LLM synthesizes an answer with source session links. Otter's AI Chat across meetings is their stickiest feature -- this brings it local-first.

### UX Polish and Delight

- [ ] **Waveform Visualization with Seek** -- Replace the simple AudioLevelBar with a rendered waveform of the full recording in PlaybackControlView. Allow click-to-seek on the waveform. Highlight the currently-playing segment. Show transcript segments as colored regions on the waveform. This is the kind of polish that makes an app feel premium and justifies a price tag -- every serious audio app (Descript, Audacity, Voice Memos) has this.

- [ ] **Live Transcript Highlight During Playback** -- During audio playback in SessionDetailView, auto-scroll the transcript and highlight the currently-spoken segment in real-time (karaoke-style sync). Click any transcript segment to seek audio to that timestamp. The timestamp data already exists in `TranscriptSegment.startTime`/`endTime` -- this just wires it to the playback position. Transforms passive reading into active audio-text navigation.

- [ ] **Keyboard-First Power User Experience** -- Add comprehensive keyboard shortcuts: Space to play/pause, arrow keys to skip 5s/15s, Cmd+E to export, Cmd+K to open command palette (Raycast-style) for quick actions (search sessions, start recording, generate summary, copy transcript). Display shortcut hints in menus and tooltips. Power users expect this from a native macOS app and it is a major differentiator vs. Electron-based competitors.

- [ ] **Session Pinboard / Favorites** -- Add a "pinned" section at the top of SessionListView for starred sessions. Quick-pin via right-click or Cmd+D. Pinned sessions persist across date filters. Simple feature that dramatically improves navigation for users with hundreds of sessions.

### Export and Integration

- [ ] **Markdown Export with YAML Frontmatter** -- Export sessions as clean Markdown files with YAML frontmatter (title, date, duration, tags) ready for Obsidian/Logseq vaults. Include transcript as timestamped text, summaries as structured sections, and action items as task lists. Offer "Export to folder" with configurable output directory (per-vault targeting). Obsidian users are the most vocal PKM community and they will champion an app that integrates cleanly with their workflow.

- [ ] **Auto-Export Pipeline** -- Configurable auto-export that triggers after each recording completes: write Markdown to a chosen folder, copy action items to clipboard, or send a webhook/URL scheme. Pairs with the Markdown export above to create a zero-friction Obsidian/Notion workflow. Shadow and Lindy built their value prop on "your meeting notes just appear where you need them."

- [ ] **Share as Rich Summary Card** -- Generate a shareable HTML or image card from a session summary (title, key points, action items, duration) that looks great when pasted into Slack, email, or docs. Use NSAttributedString rendering or SwiftUI-to-image capture. More polished than raw text paste and much more likely to be shared with teammates.

### Local-First Differentiators

- [ ] **Offline-First LLM with Bundled Model Profiles** -- Ship curated model profiles for popular local LLM setups (Ollama + llama3, LM Studio + qwen3, MLX models). Include one-click "Test Connection" in settings and a setup wizard for first-time users. Show estimated quality/speed for each model on the user's hardware. The biggest barrier to local LLM usage is configuration friction -- reducing it to one click is a competitive moat that cloud-only services cannot match.

- [ ] **Speaker Diarization Labels** -- The `TranscriptSegment.speakerLabel` field already exists but is unused. Implement basic speaker change detection using audio energy patterns and silence gaps, then let users name speakers post-recording ("Speaker 1" -> "Alice"). Display speaker labels as colored badges in the transcript. Optionally use the LLM to infer speaker names from context ("Based on the transcript, Speaker 1 appears to be the project manager..."). This is consistently the #1 requested feature in meeting transcription apps.

- [ ] **Confidential Mode with Session Encryption** -- Offer per-session encryption using a user-provided password (CryptoKit AES-GCM). Encrypted sessions show a lock icon and require the password to view. Audio files, transcripts, and summaries all encrypted at rest. No cloud service can offer this level of data sovereignty. Positions the app for sensitive use cases: legal, medical, HR, journalism, therapy notes.

### Accessibility and Inclusivity

- [ ] **Real-Time Translation Overlay** -- During live recording, offer an optional secondary panel showing the transcript translated into a user-chosen language via the LLM. Useful for multilingual meetings or language learners. Store translations alongside the original transcript. No competitor offers local real-time translation -- this is a genuine differentiator.

- [ ] **VoiceOver and Accessibility Audit** -- Conduct a full VoiceOver audit of every view. Add proper `accessibilityLabel`, `accessibilityHint`, and `accessibilityValue` to all interactive elements. Ensure the waveform and audio level meter have text alternatives. Support Dynamic Type scaling. Test with Switch Control. Most transcription apps have poor accessibility -- being best-in-class here wins institutional buyers (universities, government, healthcare).

## Round 2 — 2026-03-24 (小而美 · Small but Beautiful)

> Low-effort, high-delight features. Each should take 1-3 days max. The goal is to make users smile, feel the craft, and reduce daily micro-frustrations.

### Recording Experience

- [ ] **Pulsing Ring Recording Indicator** -- Replace the flat red dot in RecordingControlView with a concentric ripple animation (3 rings expanding outward from center, fading as they grow) that breathes with the actual audio level. Ties the visual heartbeat to real speech energy. Things 3 and Bear prove that one lovingly-animated element elevates the entire app's perceived quality. (effort: small)

- [ ] **Subtle Sound Effects on State Transitions** -- Play a soft, satisfying sound when recording starts (a warm "click-on"), pauses (gentle tone-down), resumes (rising tone), and stops (completion chime). Use `NSSound` or `AVAudioPlayer` with bundled 0.3s WAV files. Klack proved that tasteful audio feedback makes an app feel alive. Include a "Mute sounds" toggle in Recording settings. (effort: small)

- [ ] **Session Auto-Title from First Words** -- Instead of generic "Recording 2026-03-24 14:30", auto-generate the title from the first 5-8 meaningful words of the transcript once ASR delivers the first final segment (e.g., "Discussion about Q2 budget targets..."). Truncate with ellipsis. Users can still edit. Eliminates the "wall of identical timestamps" problem in the session list. (effort: tiny)

- [ ] **Elapsed Time Speaks in Human Units** -- Below the monospaced elapsed timer, show a subtle secondary label in natural language that updates at thresholds: "just started", "5 min in", "halfway there" (for scheduled recordings with known duration), "over an hour". Small text, `.secondary` color. Gives an at-a-glance sense of scale without reading numbers. (effort: tiny)

### Session Browsing & Sidebar

- [ ] **Session Preview on Hover (Quick Look)** -- Show a popover on long-hover (0.5s) over a session row in SessionListView with a 3-line transcript preview, summary snippet (first sentence of overall summary), duration, and segment count. Like Finder's Quick Look but inline. Lets users scan sessions without clicking. Use `.popover(isPresented:)` triggered by `onHover` with debounce. (effort: small)

- [ ] **Relative Date Headers with Personality** -- Replace "Mar 24, 2026" section headers in the session list with relative labels: "Today", "Yesterday", "This Week", "Last Week", "March 2026", etc. Feels warmer and more human. Calendar.current already has `isDateInToday`/`isDateInYesterday` -- just needs a formatting function. (effort: tiny)

- [ ] **Session Count Badge in Date Filter Chips** -- Add a small count badge (e.g., "Today (3)") to each DateFilterChip showing how many sessions match that filter. Prevents clicking into empty filters and gives a quick pulse-check on recording activity. Just `filteredSessions.count` per filter. (effort: tiny)

- [ ] **Swipe-to-Delete with Undo Toast** -- Instead of immediate permanent deletion, show a brief "Session deleted" toast/snackbar at the bottom of the sidebar with an "Undo" button that persists for 5 seconds. Use a soft delete (hide from list, defer `modelContext.delete` until timer expires). Prevents accidental data loss without the friction of a confirmation dialog. (effort: small)

### Transcript Reading & Typography

- [ ] **Transcript Click-to-Copy Segment** -- Single-click on any transcript segment timestamp copies that segment's text to the clipboard with a brief "Copied" tooltip animation. Double-click copies with timestamp prefix. Users constantly need to grab a specific quote -- this saves right-click > Copy dance every time. (effort: tiny)

- [ ] **Reading Progress Indicator** -- Add a thin accent-colored progress bar at the top of the transcript ScrollView that fills based on scroll position (0% at top, 100% at bottom). In long transcripts, this gives a sense of position and remaining content. GeometryReader + ScrollViewReader makes this straightforward. (effort: small)

- [ ] **Paragraph Grouping with Silence Gaps** -- When consecutive transcript segments have less than 2 seconds between them, render them as a single visual paragraph (no vertical spacing). When the gap exceeds a configurable threshold (default 3s), insert extra spacing and a subtle timestamp divider. Transforms a wall of segments into readable conversation blocks. (effort: small)

### Summary & Content Polish

- [ ] **One-Click Summary Copy** -- Add a small copy button (doc.on.doc icon) that appears on hover at the top-right corner of SummaryCardView. Copies the summary content as plain text (stripping markdown). Shows a brief checkmark animation on success. The current path is: select text > right-click > Copy. This saves 3 clicks on the most common summary action. (effort: tiny)

- [ ] **Summary Word Count & Reading Time** -- Show "~320 words, 1 min read" as a subtle caption below each summary. Uses a simple `content.split(separator: " ").count` and `wordCount / 200` formula. Gives users a sense of density before they start reading. (effort: tiny)

- [ ] **Markdown Preview Toggle in Summary** -- Add a small toggle button (eye/eye.slash) on SummaryCardView that switches between raw markdown text and rendered preview. Currently summaries render inline markdown via `AttributedString`, but users who want to copy-paste into Obsidian or Notion need to see the raw formatting. (effort: small)

### System Integration & Workflow

- [ ] **Spotlight Indexing of Sessions** -- Index sessions in Spotlight via `CSSearchableItem` so users can find recordings by typing session titles or transcript snippets in macOS Spotlight (Cmd+Space). Deep-link back to the session via `NSUserActivity`. Makes Notetaker feel like a first-class citizen of the OS. (effort: medium)

- [ ] **Global Hotkey for Quick Record** -- Register a system-wide keyboard shortcut (default: Ctrl+Opt+R, configurable) that starts/stops recording from anywhere, even when Notetaker is not frontmost. Use `CGEvent.tapCreate` or `NSEvent.addGlobalMonitorForEvents`. The menu bar icon pulses to confirm. Like Raycast's global trigger -- muscle memory in a day. (effort: medium)

- [ ] **Drag-and-Drop Session Export** -- Allow dragging a session row from the sidebar directly into Finder, Notes, or any text field. The drop payload is a formatted transcript + summary as plain text (or .md file for Finder). Uses `Transferable` protocol and `.draggable()` modifier. Zero-friction export without any menu interaction. (effort: small)

### Visual Polish & Delight

- [ ] **Empty State Illustrations** -- Replace the generic SF Symbol + text in `ContentUnavailableView` with custom, warm illustrations for each empty state: a friendly microphone character for "No sessions yet", a magnifying glass over papers for "No search results", a notepad with sparkles for "No summaries". Even simple hand-drawn SVGs dramatically raise perceived quality. Bear and Things 3 both excel here. (effort: medium)

- [ ] **Smooth State Transitions with matchedGeometryEffect** -- When a recording completes and the view switches from LiveRecordingView to SessionDetailView, use `matchedGeometryEffect` to animate the elapsed time, title, and transcript smoothly from the recording layout to the detail layout. Creates a sense of continuity instead of an abrupt cut. (effort: small)

- [ ] **Session Duration Color Coding** -- Apply a subtle left border accent color to session rows based on duration: green for quick (<5 min), blue for medium (5-30 min), purple for long (30-60 min), orange for marathon (>60 min). No legend needed -- the pattern becomes intuitive after a few sessions. Adds visual texture to the sidebar without clutter. (effort: tiny)

## Round 3 — 2026-03-24 (Deep Cuts · 深度挖掘)

> Leveraging macOS Tahoe platform capabilities, data insights, workflow automation, and novel AI applications. These are the features that make Notetaker feel like it belongs on macOS 26 and nowhere else.

### Platform-Native Intelligence (macOS 26 / Apple Intelligence)

- [ ] **Foundation Models Zero-Config Summarizer** -- Add Apple's on-device Foundation Models framework (`FoundationModels.framework`) as a new LLM engine alongside Ollama/OpenAI/Anthropic. Uses the `@Generable` macro with typed Swift structs for structured summary output (key points, action items, sentiment) via guided generation -- no API key, no server, no setup. The ~3B on-device model is free and instant on any Apple Silicon Mac running macOS 26. This eliminates the #1 onboarding barrier ("I don't have an LLM set up") and makes the app useful out of the box for the first time.

- [ ] **Foundation Models Tool Calling for Context Enrichment** -- Implement Apple's `Tool` protocol to let the on-device Foundation Model call back into the app during generation: fetch related session transcripts, look up calendar event details for the current recording, or retrieve speaker names from contacts. This makes summaries context-aware ("Alice mentioned the Q3 deadline from your March 12 product review") without sending any data off-device. No competitor has this because it requires native Swift + Apple silicon.

- [ ] **Liquid Glass Recording Dashboard** -- Redesign the LiveRecordingView control bar and session detail header using macOS Tahoe's `glassEffect(_:in:)` modifier and `GlassEffectContainer`. The recording controls float as a Liquid Glass pill over the transcript, the sidebar adopts the new floating NavigationSplitView style, and summary cards use `.glassEffect(.regular)` with the adaptive tinting. Recompiling with Xcode 26 gets automatic Liquid Glass for standard controls, but hand-tuning the custom views (AudioLevelBar, waveform, control bar) is what separates a "recompiled" app from a "designed for Tahoe" app.

- [ ] **Desktop Widget for Active Recording** -- Ship a WidgetKit Live Activity / desktop widget that shows the current recording status (elapsed time, audio level, segment count) or a summary of today's sessions on the macOS Tahoe desktop. Uses `ActivityKit` for live recording state and `WidgetKit` timeline for daily recap. macOS Tahoe finally supports Live Activities on desktop -- being one of the first apps to use this creates App Store editorial visibility.

### Shortcuts & Workflow Automation

- [ ] **App Intents for Shortcuts & Siri** -- Expose core actions via the `AppIntents` framework: "Start Recording", "Stop Recording", "Summarize Last Session", "Export Session as Markdown", "Search Transcripts for [query]". Each intent returns rich results (session title, summary snippet) that can be chained in Shortcuts or invoked via Siri. WWDC25 specifically highlighted AppIntents as the path for Shortcuts and Spotlight integration. This makes Notetaker automatable -- "Hey Siri, start a meeting recording" or a Shortcut that auto-records every Monday standup and drops notes into Obsidian.

- [ ] **AppleScript / JXA Dictionary** -- Expose a scripting dictionary (`notetaker.sdef`) for AppleScript and JavaScript for Automation (JXA) with objects for `session`, `transcript`, `summary` and verbs like `start recording`, `export session`, `get transcript of session`. Power users on macOS still rely on AppleScript for glue automation (Keyboard Maestro, Hazel, FastScripts). This is a uniquely macOS capability that web-based competitors literally cannot offer.

### Data Visualization & Analytics

- [ ] **Session Analytics Dashboard** -- Add a "Stats" tab in the sidebar showing personal recording analytics: total hours recorded this week/month/year, average session duration, recording frequency heatmap (GitHub-style contribution grid), busiest recording days/times, word count trends, and summary generation count. Uses SwiftUI Charts framework. Gives users a sense of their recording habits and makes the app feel like it's learning about them. Toggl and RescueTime proved that personal analytics create habitual engagement.

- [ ] **Transcript Vocabulary Heatmap** -- After summarization, generate a word frequency analysis of the session transcript and display it as an interactive word cloud or ranked term list in SessionDetailView. Highlight domain-specific jargon, proper nouns, and recurring themes across sessions. Useful for researchers, journalists, and students who want to see "what did we talk about most?" at a glance. Pure local computation, no LLM needed.

### Advanced Audio

- [ ] **Noise-Aware Recording Quality Indicator** -- During live recording, analyze the audio signal in real-time (already computing RMS in `AudioCaptureService`) to display a quality badge: "Excellent" (clean speech), "Good" (minor background noise), "Noisy" (significant interference), "Poor" (mostly noise). Show as a subtle colored dot next to the audio level meter. Warn users when quality drops below threshold with a gentle notification ("Try moving closer to the microphone"). Krisp proved that audio quality awareness improves outcomes -- but instead of filtering noise (which degrades speech for ASR), just inform the user so they can fix the source.

- [ ] **Audio Bookmarks with Inline Notes** -- Let users tap a "bookmark" button (or press B) during live recording to drop a timestamped marker. Each bookmark can have an optional short text note ("important decision here", "revisit this"). Display bookmarks as colored pins on the playback timeline and as a clickable list in SessionDetailView. Fireflies offers clip-and-share for key moments -- this is the local-first equivalent, and it's useful during lectures, interviews, and long meetings where you want to flag moments without stopping to type.

### Content Organization

- [ ] **Smart Collections with Saved Predicates** -- Beyond simple tag filtering, let users create Smart Collections (like Smart Playlists in Music.app or Smart Mailboxes in Mail.app): "All sessions longer than 30 minutes", "Sessions this month with summaries containing 'budget'", "Untagged sessions from last week". Store as serialized `NSPredicate` / SwiftData `#Predicate` configurations. Display as expandable sections in the sidebar below pinned sessions. This is a deeply macOS-native concept that power users expect from serious data management apps.

- [ ] **Session Linking and Related Notes** -- Allow users to manually link related sessions together (e.g., "Weekly Standup Mar 17" <-> "Weekly Standup Mar 24") or have the LLM suggest links based on transcript similarity. Display linked sessions as a "Related" section in SessionDetailView. Enables narrative threads across recording sessions -- following a project arc, a recurring meeting series, or a multi-part interview. Obsidian's backlinks proved that relationship graphs transform isolated notes into a knowledge network.

### Personalization & Theming

- [ ] **Custom Accent Color and Sidebar Tint** -- Let users pick a custom accent color (beyond the system accent) for the app's UI elements: recording indicator, summary card borders, active session highlight, and Liquid Glass tint. Store in `@AppStorage`. Offer 8 curated presets (Ocean, Forest, Sunset, Monochrome, etc.) plus a custom color picker. Small but surprisingly impactful -- Bear, Things 3, and Craft all offer this and users love it. On Tahoe, the Liquid Glass material adapts to the accent color, making the entire app feel personalized.

- [ ] **Custom Word Bank for ASR Accuracy** -- Let users maintain a list of custom terms, proper nouns, and domain jargon (e.g., "Kubernetes", "EBITDA", "Dr. Ramirez") that the app displays alongside transcripts for easy correction. When using Foundation Models for post-processing, inject the word bank into the prompt to improve proper noun rendering in summaries. Jamie charges for this as a premium feature ("word banks") -- offering it free in a local-first app is a differentiator.

### Performance & Efficiency

- [ ] **Incremental Transcript Index for Instant Search** -- Build a local full-text search index (using `SearchKit.framework` or SQLite FTS5 via SwiftData's backing store) that updates incrementally as new transcript segments arrive. Current search scans SwiftData relationships which triggers faulting storms on large session counts. An indexed approach makes search across thousands of sessions sub-100ms. SearchKit is an Apple framework that web apps cannot use, and it supports ranked relevance, phrase matching, and boolean queries out of the box.

- [ ] **Background Audio Processing with Service Management** -- Register a `SMAppService` login item that can perform deferred work: re-index transcripts, generate missing summaries for sessions that were recorded while the LLM was unavailable, run batch auto-tagging overnight, and pre-warm the LLM connection. Uses `BGProcessingTaskRequest`-style scheduling on macOS. Transforms the app from "active-only" to "always working for you in the background" -- like how Photos.app processes ML classification overnight.

## Round 4 — 2026-03-24 (Frontend Craft / 前端工艺)

> Inspired by frontend-design aesthetics: typography, color, motion, spatial composition, and visual details. Each item is a small, handcrafted polish that elevates perceived quality. These are the details users feel but can't articulate -- the difference between "it works" and "it feels right."

### Typography & Reading Experience

- [ ] **Monospaced Tabular Figures for All Timestamps** -- Replace `.system(.caption, design: .monospaced)` timestamp font with `.monospacedDigit()` applied to the existing caption style. Full monospaced text makes letters like "m" and "i" unnaturally wide; `.monospacedDigit()` keeps proportional letter spacing but aligns digits so timestamps like `03:42` and `12:07` share identical column width. Eliminates subtle jitter when transcript segments scroll in during live recording. One line change per usage site in `DS.Typography`. (effort: tiny)

- [ ] **Optical Text Weight Balancing in Session Headers** -- The session detail header currently uses uniform `.semibold` for the title and `.callout` for metadata. Refine to a three-level typographic hierarchy: title in `.title2.weight(.medium)` with slightly increased tracking (`.tracking(0.3)`), date/duration in `.subheadline` with `.foregroundStyle(.secondary)`, and segment/word count in `.caption` with `.foregroundStyle(.tertiary)`. The visual weight decreases smoothly down the hierarchy, guiding the eye. Inspired by Craft's session headers. (effort: tiny)

- [ ] **Fade-to-Truncation for Long Transcript Segments** -- When a transcript segment in `TranscriptSegmentRow` exceeds 4 lines, instead of hard `.lineLimit(4)` truncation with "...", overlay a linear gradient mask (`LinearGradient` from `.clear` to `.background`) at the bottom edge. Tap to expand. Creates a soft visual cue that content continues without the jarring clip of text truncation. Bear uses this pattern for long note previews in its sidebar. (effort: small)

- [ ] **Contextual Line Spacing in Summaries** -- Apply `.lineSpacing(4)` and `.kerning(0.2)` to summary content in `SummaryCardView` for improved readability of LLM-generated prose. Currently uses raw `.callout` font defaults which are optimized for UI labels, not for reading paragraphs of content. The subtle spacing increase makes summaries feel like a crafted document rather than debug output. Add `DS.Typography.prose` as a new token. (effort: tiny)

### Color & Adaptive Theming

- [ ] **Audio Level Gradient with Contextual Color** -- Replace the flat `DS.Colors.audioLevel` (solid green) in `AudioLevelBar` with a horizontal gradient that shifts from green (low) through yellow (mid) to orange (high), mapped to the `level` float. Use `LinearGradient(stops:)` with three color stops. The bar becomes an intuitive signal quality meter: calm green means normal speech, orange flash means shouting or clipping. One `fill()` change. (effort: tiny)

- [ ] **Summary Style Color Accents** -- Assign each `SummaryStyle` (bullet, narrative, detailed) a subtle left border color: indigo for bullet, teal for narrative, amber for detailed. Render as a 3px leading rounded rectangle in `SummaryCardView` via `.overlay(alignment: .leading)`. At a glance, users can distinguish summary types without reading labels. Inspired by Notion's database color-coding. (effort: tiny)

- [ ] **Time-of-Day Adaptive Empty State** -- In the idle recording view (big red button state), show a subtle contextual greeting that adapts to the time of day: "Good morning" / "Good afternoon" / "Good evening" in `.title3.weight(.light)` with `.foregroundStyle(.quaternary)` above the record button. Uses `Calendar.current.component(.hour, from: Date())`. Makes the empty state feel alive instead of static. Disappears the moment recording starts. (effort: tiny)

### Motion & Micro-interactions

- [ ] **Spring-Loaded Button Feedback** -- Replace `.buttonStyle(.plain)` on the main record, pause, stop, and play buttons with a custom `ButtonStyle` that applies `.scaleEffect(isPressed ? 0.88 : 1.0)` with `.spring(response: 0.25, dampingFraction: 0.6)`. The buttons physically compress on press and bounce back on release. Costs 15 lines for a `PressableButtonStyle` struct in `ViewModifiers.swift`, reusable across the entire app. Klack and Things 3 both use this pattern. (effort: tiny)

- [ ] **Transcript Segment Slide-In Animation** -- When new transcript segments arrive during live recording, animate them in with `.transition(.asymmetric(insertion: .move(edge: .bottom).combined(with: .opacity), removal: .opacity))` and `.animation(.spring(response: 0.4, dampingFraction: 0.8))`. Currently segments just appear instantly. The slide-in creates a sense of flow -- words arriving into the view like speech arriving in real time. (effort: tiny)

- [ ] **Smooth Audio Level Bar with Spring Physics** -- Change `AudioLevelBar`'s animation from `.linear(duration: 0.05)` to `.spring(response: 0.15, dampingFraction: 0.7)`. Linear animation creates a mechanical feel; spring animation makes the bar feel organic and alive, like a physical VU meter needle that overshoots slightly and settles. The bar becomes something users enjoy watching. One parameter change. (effort: tiny)

- [ ] **Copy Confirmation Micro-animation** -- When users copy transcript or summary text, show a brief inline confirmation: the copy button icon morphs from `doc.on.doc` to `checkmark` with `.contentTransition(.symbolEffect(.replace))` and reverts after 1.5 seconds via `Task.sleep`. Eliminates the "did it copy?" uncertainty. SwiftUI's SF Symbol animation API makes this trivial with `.symbolEffect(.bounce)`. (effort: tiny)

### Spatial Composition & Layout

- [ ] **Asymmetric Detail View with Weighted Panels** -- In `SessionDetailView`, when both transcript and overall summary are visible, use a weighted split: 62% transcript, 38% summary panel. The current 50/50 split gives equal visual weight to unequal content (transcript is always longer). The asymmetry prioritizes the primary content (transcript) while keeping the summary visible as a reference pane. Implement with `GeometryReader` percentage-based widths. (effort: small)

- [ ] **Breathing Negative Space in Transcript** -- Add `DS.Spacing.lg` (16pt) horizontal padding to the transcript text column (not the timestamp), and increase inter-segment spacing from `DS.Spacing.xxs` (2pt) to `DS.Spacing.sm` (8pt). The current tight spacing makes long transcripts feel dense and tiring. The additional whitespace creates a comfortable reading rhythm, especially for sessions over 30 minutes. Craft and iA Writer both prioritize generous text margins. (effort: tiny)

- [ ] **Collapsible Control Bar with Peek State** -- When scrolling down the transcript in `SessionDetailView`, auto-collapse the playback controls to a compact single-line peek (just play/pause, current time, and a thin progress bar) using `.safeAreaInset(edge: .bottom)` with an animated height transition. Scrolling up or hovering restores full controls. Maximizes transcript reading space while keeping playback always accessible. Inspired by Safari's collapsing toolbar. (effort: small)

### Visual Details & Atmosphere

- [ ] **Subtle Card Elevation on Hover** -- Add `.shadow(color: .black.opacity(isHovered ? 0.08 : 0.03), radius: isHovered ? 8 : 2, y: isHovered ? 3 : 1)` with `.animation(.easeOut(duration: 0.2))` to the `.cardStyle()` modifier in `ViewModifiers.swift`. Summary cards and session rows gently lift when hovered, creating a tactile sense of depth. Combined with a slight `.scaleEffect(isHovered ? 1.005 : 1.0)`, the card feels like it responds to the cursor. Small enough to be subliminal, impactful enough to feel premium. (effort: tiny)

- [ ] **Recording State Ambient Glow** -- During active recording, add a very subtle red radial gradient glow (`.background(RadialGradient(...)).opacity(0.04)`) behind the entire `LiveRecordingView` that pulses gently with the existing `pulseAnimation` state. The glow gives the whole view a warm "alive" feeling without being distracting. Opacity is low enough (3-5%) that users feel it subconsciously rather than see it explicitly. Stop recording and it fades away. (effort: tiny)

Store custom bindings in `@AppStorage("customShortcuts")` as a JSON dictionary mapping action identifiers to key+modifier tuples. The app's `.keyboardShortcut()` modifiers read from this dictionary with fallback to defaults. A "Reset to Defaults" button restores all bindings. Uses the `sindresorhus/KeyboardShortcuts` pattern of a recorder view, but implemented natively with SwiftUI's `onKeyPress` for in-app shortcuts (not global). Power users on macOS expect to remap shortcuts to match their muscle memory from other apps -- Xcode, Terminal, and every serious Mac app offers this. Different from "Keyboard Shortcut Cheat Sheet" (Round 9) which *displays* shortcuts and "Discoverable Shortcut Hints" (Round 9) which *teaches* them -- this lets users *change* them. (effort: medium)
