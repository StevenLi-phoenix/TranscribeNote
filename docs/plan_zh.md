### 高级音频

- [ ] **噪声感知的录音质量指示器** -- 在实时录音期间，分析音频信号（已在 `AudioCaptureService` 中计算 RMS）以显示质量标签："优秀"（干净语音）、"良好"（轻微背景噪音）、"嘈杂"（明显干扰）、"较差"（大部分为噪音）。在音频电平表旁显示微妙的彩色点。质量低于阈值时温和通知用户（"试试靠近麦克风"）。Krisp 证明了音频质量感知能改善结果——但不是过滤噪音（会降低 ASR 的语音质量），而是告知用户让其修复源头。

- [ ] **带内联备注的音频书签** -- 让用户在实时录音期间点击 "书签" 按钮（或按 B）插入带时间戳的标记。每个书签可附带可选的简短文字备注（"重要决定在这里"、"回头再看这段"）。在播放时间线上显示为彩色图钉，在 SessionDetailView 中显示为可点击列表。Fireflies 提供关键时刻的剪辑分享——这是本地优先的等价物。

### 内容组织

- [ ] **智能集合（保存的谓词查询）** -- 超越简单的标签筛选，让用户创建智能集合（类似 Music.app 的智能播放列表或 Mail.app 的智能邮箱）："所有超过 30 分钟的会话"、"本月包含'预算'摘要的会话"、"上周未标记的会话"。存储为序列化的 `NSPredicate` / SwiftData `#Predicate` 配置。在侧栏中置顶会话下方显示为可展开区域。这是高级用户对严肃数据管理应用的期望，是深度 macOS 原生概念。

- [ ] **会话关联与相关笔记** -- 允许用户手动关联相关会话（如 "周会 3月17日" <-> "周会 3月24日"），或让 LLM 基于转录相似度建议关联。在 SessionDetailView 中显示 "相关" 区域。支持跨录音会话的叙事线索——追踪项目弧线、定期会议系列或多部分访谈。Obsidian 的反向链接证明，关系图将孤立笔记变成知识网络。

### 个性化与主题

- [ ] **自定义强调色与侧栏色调** -- 让用户选择自定义强调色（超越系统强调色）用于应用的 UI 元素：录音指示器、摘要卡片边框、活跃会话高亮和 Liquid Glass 色调。存储在 `@AppStorage` 中。提供 8 个精选预设（海洋、森林、日落、单色等）加自定义取色器。效果出乎意料——Bear、Things 3 和 Craft 都提供这个，用户很喜欢。在 Tahoe 上，Liquid Glass 材质会适配强调色，让整个应用感觉个性化。

- [ ] **自定义词库提升 ASR 准确度** -- 让用户维护自定义术语、专有名词和专业行话列表（如 "Kubernetes"、"EBITDA"、"张医生"），应用在转录旁显示以便纠正。使用 Foundation Models 做后处理时，将词库注入提示词以改善摘要中专有名词的呈现。Jamie 将此作为付费功能（"word banks"）——在本地优先应用中免费提供就是差异化。

### 性能与效率

- [ ] **增量转录索引实现即时搜索** -- 构建本地全文搜索索引（使用 `SearchKit.framework` 或 SwiftData 底层的 SQLite FTS5），随新转录片段到达增量更新。当前搜索扫描 SwiftData 关系会在大量会话时触发 faulting storm。索引方式让跨数千会话搜索控制在 100ms 以内。SearchKit 是 Apple 框架，Web 应用无法使用，它开箱支持排名相关度、短语匹配和布尔查询。

- [ ] **后台音频处理与服务管理** -- 注册 `SMAppService` 登录项执行延迟工作：重新索引转录、为 LLM 不可用时录制的会话生成缺失摘要、夜间批量自动标记、预热 LLM 连接。使用 macOS 上的 `BGProcessingTaskRequest` 风格调度。将应用从 "仅活跃时工作" 变成 "始终在后台为你工作"——类似 Photos.app 在夜间处理 ML 分类。

## 第四轮 — 2026-03-24（前端工艺）

> 灵感来自前端设计美学：排版、色彩、动效、空间构成和视觉细节。每项都是小巧手工打磨，提升感知品质。这些是用户感受到但无法言说的细节——"能用" 和 "感觉对了" 之间的差距。

### 排版与阅读体验

- [ ] **时间戳使用等宽数字** -- 将 `.system(.caption, design: .monospaced)` 时间戳字体替换为在现有 caption 样式上应用 `.monospacedDigit()`。全等宽文本让 "m" 和 "i" 等字母不自然地宽；`.monospacedDigit()` 保持比例字母间距但对齐数字，使 `03:42` 和 `12:07` 等时间戳共享相同列宽。消除实时录音期间转录片段滚入时的微妙抖动。`DS.Typography` 中每个使用位置改一行。（工作量：极小）

- [ ] **会话头部的光学字重平衡** -- 当前会话详情头部标题统一使用 `.semibold`，元数据使用 `.callout`。优化为三级排版层次：标题 `.title2.weight(.medium)` 配合略增的字距（`.tracking(0.3)`），日期/时长 `.subheadline` 配合 `.foregroundStyle(.secondary)`，片段/字数 `.caption` 配合 `.foregroundStyle(.tertiary)`。视觉重量沿层级平滑递减，引导视线。灵感来自 Craft 的会话头部。（工作量：极小）

- [ ] **长转录片段的渐隐截断** -- 当 `TranscriptSegmentRow` 中的转录片段超过 4 行时，不使用硬性 `.lineLimit(4)` 截断加 "..."，而是在底边叠加线性渐变遮罩（`LinearGradient` 从 `.clear` 到 `.background`）。点击展开。创造柔和的视觉提示表示内容继续，而非生硬的文本截断。Bear 在侧栏长笔记预览中使用此模式。（工作量：小）

- [ ] **摘要的上下文行距** -- 在 `SummaryCardView` 的摘要内容上应用 `.lineSpacing(4)` 和 `.kerning(0.2)` 以改善 LLM 生成文本的可读性。当前使用原始 `.callout` 字体默认值，它们是为 UI 标签优化的，不适合阅读段落内容。微妙的间距增加让摘要感觉像精心制作的文档而非调试输出。添加 `DS.Typography.prose` 作为新 token。（工作量：极小）

### 色彩与自适应主题

- [ ] **音频电平渐变与语境色彩** -- 将 `AudioLevelBar` 中平坦的 `DS.Colors.audioLevel`（纯绿）替换为水平渐变，从绿色（低）经黄色（中）到橙色（高），映射到 `level` 浮点值。使用 `LinearGradient(stops:)` 配合三个色标。音量条变成直觉的信号质量计：平静绿色表示正常说话，橙色闪现表示大喊或削波。一个 `fill()` 更改。（工作量：极小）

- [ ] **摘要风格的颜色强调** -- 为每种 `SummaryStyle`（要点、叙述、详细）分配微妙的左侧边框色：靛蓝色表示要点，青色表示叙述，琥珀色表示详细。在 `SummaryCardView` 中通过 `.overlay(alignment: .leading)` 渲染 3px 左侧圆角矩形。一眼即可区分摘要类型而无需阅读标签。灵感来自 Notion 的数据库颜色编码。（工作量：极小）

- [ ] **随时间变化的空状态问候** -- 在空闲录音视图（大红按钮状态）中，在录音按钮上方显示微妙的随时间变化的问候语："早上好" / "下午好" / "晚上好"，使用 `.title3.weight(.light)` 配合 `.foregroundStyle(.quaternary)`。使用 `Calendar.current.component(.hour, from: Date())`。让空状态感觉有生命而非静态。录音开始时消失。（工作量：极小）

### 动效与微交互

- [ ] **弹簧按钮反馈** -- 将主录音、暂停、停止和播放按钮的 `.buttonStyle(.plain)` 替换为自定义 `ButtonStyle`，应用 `.scaleEffect(isPressed ? 0.88 : 1.0)` 配合 `.spring(response: 0.25, dampingFraction: 0.6)`。按钮按下时物理压缩，释放时弹回。只需 15 行代码在 `ViewModifiers.swift` 中写一个 `PressableButtonStyle` 结构体，可在整个应用复用。Klack 和 Things 3 都使用此模式。（工作量：极小）

- [ ] **转录片段滑入动画** -- 实时录音期间新转录片段到达时，使用 `.transition(.asymmetric(insertion: .move(edge: .bottom).combined(with: .opacity), removal: .opacity))` 和 `.animation(.spring(response: 0.4, dampingFraction: 0.8))` 添加入场动画。当前片段只是瞬间出现。滑入创造流动感——文字进入视野如同语音实时到达。（工作量：极小）

- [ ] **弹簧物理的平滑音频电平条** -- 将 `AudioLevelBar` 的动画从 `.linear(duration: 0.05)` 改为 `.spring(response: 0.15, dampingFraction: 0.7)`。线性动画产生机械感；弹簧动画让条形感觉有机和鲜活，像物理 VU 表针略微过冲然后稳定。条形变成用户喜欢观看的东西。改一个参数。（工作量：极小）

- [ ] **复制确认微动画** -- 用户复制转录或摘要文本时，显示简短的内联确认：复制按钮图标从 `doc.on.doc` 变形为 `checkmark`，使用 `.contentTransition(.symbolEffect(.replace))`，1.5 秒后通过 `Task.sleep` 恢复。消除 "复制成功了吗？" 的不确定性。SwiftUI 的 SF Symbol 动画 API 使用 `.symbolEffect(.bounce)` 即可实现。（工作量：极小）

### 空间构成与布局

- [ ] **非对称详情视图与加权面板** -- 在 `SessionDetailView` 中，当转录和总体摘要同时可见时，使用加权分割：62% 转录，38% 摘要面板。当前 50/50 分割给不等量内容（转录总是更长）赋予了相同的视觉权重。非对称性优先展示主要内容（转录），同时保持摘要作为参考面板可见。使用 `GeometryReader` 百分比宽度实现。（工作量：小）

- [ ] **转录中的呼吸留白** -- 为转录文本列（非时间戳）添加 `DS.Spacing.lg`（16pt）水平内边距，并将片段间距从 `DS.Spacing.xxs`（2pt）增加到 `DS.Spacing.sm`（8pt）。当前紧密间距让长转录感觉密集且疲劳。额外留白创造舒适的阅读节奏，尤其是超过 30 分钟的会话。Craft 和 iA Writer 都优先考虑宽裕的文本边距。（工作量：极小）

- [ ] **可折叠控制栏与窥视状态** -- 在 `SessionDetailView` 中向下滚动转录时，自动折叠播放控制为紧凑的单行窥视（仅播放/暂停、当前时间和细进度条），使用 `.safeAreaInset(edge: .bottom)` 配合动画高度过渡。向上滚动或悬停恢复完整控制。最大化转录阅读空间，同时保持播放始终可访问。灵感来自 Safari 的折叠工具栏。（工作量：小）

### 视觉细节与氛围

- [ ] **悬停时微妙的卡片提升** -- 在 `ViewModifiers.swift` 的 `.cardStyle()` 修饰符中添加 `.shadow(color: .black.opacity(isHovered ? 0.08 : 0.03), radius: isHovered ? 8 : 2, y: isHovered ? 3 : 1)` 配合 `.animation(.easeOut(duration: 0.2))`。摘要卡片和会话行悬停时轻轻抬起，创造触感深度。配合轻微 `.scaleEffect(isHovered ? 1.005 : 1.0)`，卡片感觉在响应光标。小到潜意识，但影响足以感到高级。（工作量：极小）

- [ ] **录音状态环境光晕** -- 活跃录音期间，在整个 `LiveRecordingView` 背后添加非常微妙的红色径向渐变光晕（`.background(RadialGradient(...)).opacity(0.04)`），随现有 `pulseAnimation` 状态轻柔脉动。光晕给整个视图温暖的 "活跃" 感觉而不分散注意力。透明度足够低（3-5%）使用户潜意识感受而非明确看到。停止录音后淡出。（工作量：极小）

- [ ] **会话行状态图标点** -- 在 `SessionListView` 每个会话行添加 6pt 圆圈，使用渐变填充反映会话状态：纯绿点表示完整会话，半填充琥珀色表示部分/中断会话，蓝色圆环表示有摘要的会话。定位在标题左侧如状态图标。给会话列表视觉纹理和可扫描的状态指示器，无需文本标签。（工作量：小）

- [ ] **内容区域间的毛玻璃分隔线** -- 将头部、播放控制和转录区域之间的裸 `Divider()` 替换为自定义分隔线：1px 线条配合 `.background(.ultraThinMaterial)` 和 16pt 水平内缩（非边到边）。毛玻璃材质拾取下方内容颜色，使分隔线感觉融入而非强加。小细节区分 "默认 SwiftUI" 和 "为 Tahoe 设计"。（工作量：极小）

## 第五轮 — 2026-03-24（新功能）

> 真正的新能力，扩展应用的功能边界。聚焦于解决真实问题和创造 "啊哈" 时刻。参考 2025-2026 年会议智能、PKM、代理式 AI 和 macOS Tahoe 平台能力的趋势。

### AI 智能与理解

- [ ] **逐说话人情感时间线** -- 在有说话人分离标签的会话后，对每个说话人的片段运行情感分析，生成逐说话人的情感弧线。在 SessionDetailView 中显示为紧凑的迷你折线图（每个说话人一条线，正面/中性/负面着色）。用户一眼看出谈判中何时紧张升级、头脑风暴中何时能量下降、辅导中何时学生脱离。使用 LLM 配合分块转录窗口——无需外部 API。自然搭配说话人分离功能（第一轮）作为下游分析层。（工作量：中等）

- [ ] **自动主题分段与章节** -- 分析转录以检测主题边界，生成 "章节" 时间线（类似 YouTube 章节），带自动生成的标题。在 SessionDetailView 中显示为可点击的章节列表，跳转到对应音频时间戳。60 分钟会议变得几秒内可导航："00:00 介绍"、"04:30 Q2营收回顾"、"18:20 招聘计划"、"35:00 开放讨论"。使用 LLM 从转录内容和时间戳间隔识别主题切换。Granola 和 Fathom 都在发布——2026 年这是预期功能。（工作量：中等）

- [ ] **关键引用提取与归属** -- 摘要生成后，自动识别转录中 3-5 条显著的直接引用——最有影响力、决定性或令人惊讶的陈述——并显示为带说话人归属和时间戳的引用卡片。用户可点击听原始音频。适用于记者捕捉采访金句、经理提取承诺（"我周五前完成原型"）、学生摘录教授定义。没有竞品如此清晰地呈现个别引用。（工作量：小）

- [ ] **后续会话简报** -- 开始新录音时，可选显示一张 "简报卡片"，总结相关前序会话的上下文。如果日历事件标题匹配定期会议（"每周站会"），自动拉取上一次的待办事项和未解决问题。如果该主题存在智能集合，拉取最新会话的摘要。用户带着充分准备走进每次会议。Otter 的 "会前简报" 是其最具粘性的企业功能——这里将其本地化。（工作量：中等）

### 音频处理与采集

- [ ] **系统音频捕获（环回录音）** -- 使用 macOS 的 `ScreenCaptureKit`（`SCStream` 纯音频配置）录制系统音频输出和麦克风输入。在 Zoom/Teams/FaceTime 通话中捕获对方的声音，无需机器人加入会议。Granola 的方式——在设备本地捕获音频，无会议机器人，无云端。需要屏幕录制权限。这是会议笔记领域最受欢迎的功能类别，也是 Granola 融资 4300万美元的原因。（工作量：大）

- [ ] **音频片段分享（音摘导出）** -- 在播放波形上选择时间范围（或选择转录片段），仅导出那段音频切片为独立 M4A 文件。适用于从 45 分钟会议中分享特定 30 秒决定、摘取播客高光、或保存记忆声音笔记。使用 `AVAssetExportSession` 配合 `timeRange`。搭配关键引用提取——每张引用卡片可有 "导出片段" 按钮。（工作量：小）

- [ ] **多麦克风源选择** -- 让用户选择使用哪个音频输入设备（内置麦克风、AirPods、外部 USB 麦克风、音频接口），在录音设置中提供下拉菜单或录音控制栏中的快速选择器。显示设备名称和每个可用输入的实时音频电平预览。当前 `AudioCaptureService` 使用默认输入设备——这添加 `AVAudioSession.availableInputs` 枚举和 `setPreferredInput()`。使用 USB 麦克风或音频接口的专业用户期望这个。（工作量：小）

### 内容管理与组织

- [ ] **带预填元数据的会话模板** -- 让用户创建可复用的会话模板，预填标题格式、标签、关联智能集合、摘要风格和 AI 配方。模板可按模式关联日历事件（"1:1 with *" 使用 "1对1" 模板）。将每次会话设置减少到零点击。不同于 AI 配方（第一轮，提示词模板）——这是元数据 + 工作流模板。（工作量：中等）

- [ ] **带学习的转录纠错模式** -- 添加专门的 "纠错" 模式，用户可点击任意转录片段修复 ASR 错误（专有名词、术语、听错的词）。纠正存储为个人词典，应用用来后处理未来转录——如果 "Kubernetes" 曾从 "库伯内蒂斯" 被纠正，以后自动纠正。不同于自定义词库（第三轮，给 LLM 提示词的静态列表）——这是随时间改善 ASR 输出的主动学习循环。（工作量：中等）

- [ ] **会话合并与拆分** -- 允许用户合并两个或多个会话为一个组合会话（意外因暂停/重启而拆分的会议）或在选定时间戳处拆分一个会话为两个（一次录音涵盖两个不同主题）。合并通过 `AudioExporter` 组合音频片段、连接调整时间戳的转录片段并重新运行摘要。拆分在分割点之后创建新会话。（工作量：中等）

### 工作流与自动化

- [ ] **待办同步到提醒事项** -- 将提取的待办事项（来自第一轮的自动待办）直接导出到 Apple 提醒事项，通过 `EventKit` 的 `EKReminder` API。每个待办变成一个提醒，带截止日期、链接回源会话的备注和原始转录上下文。双向同步：在提醒事项中标记完成也在 Notetaker 中标记完成。使用现有的 `personal-information.calendars` 权限（EKEventStore 也处理提醒事项）。云端竞品无法匹配的原生 macOS 集成。（工作量：中等）

- [ ] **Webhook 与 URL Scheme 触发器** -- 定义会话完成时触发的自定义触发器：HTTP POST 到 webhook URL（用于 Zapier/Make/n8n 集成），或打开 URL scheme（用于 Shortcuts 链接）。Webhook 载荷包含会话标题、摘要、待办事项、标签和时长的 JSON。实现 "录音完成 → Slack 频道通知"、"标记为'招聘'的会话 → 添加到 Airtable"、或 "任何会话 → 通过 URL scheme 创建 Obsidian 笔记"。自动导出流水线（第一轮）处理文件导出——这处理事件驱动的集成。（工作量：小）

- [ ] **独立循环自动录音规则** -- 超越日历关联的计划录音，让用户定义独立的循环规则："每个工作日 9:00-9:30 录音" 或 "每周一下午 2:00 录音 60 分钟"。规则独立于日历事件，即使没有匹配事件也会触发。适用于有不在日历中的固定会议的用户，或想要每天固定时间做语音日记的人。基于现有 `SchedulerViewModel` 基础设施但与 `CalendarService` 解耦。（工作量：小）

### 数据智能与洞察

- [ ] **个人说话模式报告** -- 分析用户跨会话的说话模式：每分钟字数、填充词频率（"嗯"、"啊"、"就是"、"然后"）、平均说/听比例（有说话人分离时）、词汇丰富度分数和最长独白时长。在分析仪表盘（第三轮）中显示为月度报告卡。演讲教练每小时收费 200 美元——应用从已有数据中免费提供这些反馈。（工作量：中等）

- [ ] **自动生成会议纪要** -- 从会话生成正式 "会议纪要" 文档：参会者（来自说话人标签）、议程项目（来自主题章节）、每个议程项的讨论摘要、做出的决定、带负责人的待办事项和下次会议日期（来自日历）。导出为格式化的 PDF 或 Markdown 文件，可直接发给利益相关者。不同于现有摘要（自由散文）——这是组织机构要求的结构化专业文档格式。（工作量：中等）

- [ ] **会话差异/进度追踪** -- 对于定期会议（通过标题模式或日历链接检测），自动生成 "差异" 显示上次以来的变化：新待办 vs 结转待办、这次讨论但上次没讨论的主题、情感变化和先前提到的交付物进展。在会话详情顶部显示为 "新变化" 卡片。将应用从录音工具变成项目进度追踪器。（工作量：大）

### 平台集成

- [ ] **Handoff 到 iPhone / iPad 延续** -- 实现 `NSUserActivity` Handoff，用户可以在 Mac 上开始浏览会话转录，然后在 iPhone 或 iPad 上继续阅读（当 iOS 伴侣应用发布时）。即使 iOS 应用不存在，注册用户活动也能实现其他设备上的 Spotlight 索引和未来 Handoff。`NSUserActivity` 基础设施现在添加很轻量，以后会有回报。（工作量：小）

- [ ] **Finder 快速操作导入音频** -- 通过 `NSExtensionContext` 注册 Finder 快速操作（Services 菜单），让用户在 Finder 中右键任意音频文件选择 "用 Notetaker 转录"。应用导入文件、创建新会话，并运行 ASR + 摘要。将 Notetaker 从 "仅限实时录音" 变成通用转录工具（播客、语音备忘录、采访录音、课堂录音）。这就是 MacWhisper 等基于 Whisper 的桌面应用的差异化方式。（工作量：中等）

- [ ] **Focus 滤镜集成** -- 实现 `FocusFilterIntent` API，让用户配置不同 Focus 模式下 Notetaker 的行为。"工作" 专注时：自动开始计划录音并显示通知。"个人" 专注时：抑制录音通知。"勿扰" 时：完全禁用自动开始。让应用尊重用户的注意力状态——一个表明 macOS 原生工艺的细节。（工作量：小）

## 第六轮 — 2026-03-24（交互与智能）

> 位于精美设计与智能功能交叉点的小功能。每个都让应用感觉更有生命力、更有感知力、更令人愉悦。聚焦于新颖的交互模式、上下文智能、包容性设计和 macOS Tahoe 原生的内容导航方式。

### 引导与首次运行体验

- [ ] **带实时预览的渐进式设置向导** -- 用 3 步引导流程替换当前的冷启动体验（空会话列表 + 隐私披露弹窗）：(1) "录音"——用户录制 10 秒测试并看到实时转录出现，证明应用立即可用；(2) "总结"——测试转录通过配置的 LLM 进行摘要，端到端展示管线；(3) "准备就绪"——解释键盘快捷键和菜单栏访问。每步是全出血卡片，使用 `TabView` 配合 `.tabViewStyle(.page)` 和弹簧过渡。关键洞察：让用户先*体验*价值，再要求他们配置任何东西。始终可见跳过按钮。存储完成状态于 `@AppStorage("hasCompletedOnboarding")`。（工作量：中等）

- [ ] **上下文功能提示与引导标记** -- 不是提前解释功能，而是在用户*首次*遇到 UI 元素时显示小型非模态提示弹出窗口：暂停按钮上的提示（"暂停创建新音频片段——随时恢复"），摘要卡片上的提示（"点击编辑，或右键用自定义指令重新生成"），侧栏日期筛选器上的提示（"按时间段筛选会话"）。使用 SwiftUI 的 `.popoverTip()` 配合 TipKit 框架（macOS 14+）。每个提示只显示一次，可关闭。在调试中尊重 `TipKit.Tips.showAllTipsForTesting`。应用在用户探索时自我教学——无需手册或帮助页面。（工作量：小）

### 转录交互与导航

- [ ] **长转录的迷你地图滚动导航器** -- 对超过 15 分钟的会话，在转录 ScrollView 的尾侧显示窄型垂直迷你地图轨道（40px 宽），渲染整个转录的压缩排版密度图。密集区域（大量语音）显示更深；静音间隔显示为空白。点击迷你地图任意位置跳转。当前滚动位置显示为高亮的视窗矩形。灵感来自代码编辑器（Sublime Text、VS Code），但应用于对话。使用 `Canvas` 视图渲染映射到迷你地图高度的片段位置。在一小时长的转录中提供空间感知，标准滚动条无法告诉你内容信息。（工作量：中等）

- [ ] **转录文本选择与音频范围** -- 当用户使用标准 macOS 文本选择（点击拖动）跨多个转录片段选择文本时，显示浮动工具栏（类似系统文本选择工具栏），带 "播放选中部分" 按钮，仅播放那些片段 `startTime...endTime` 覆盖的音频范围。使用 `AVAudioPlayer.play(atTime:)` 配合计算的偏移量和在结束时间戳停止的计时器。将转录变成直接的音频导航面——点击一个词就能听到它被说出来。Descript 开创了这种 "文本即音频" 范式；这将其带入原生 SwiftUI 应用。（工作量：中等）

- [ ] **Cmd+G "跳转到时间" 快速跳转** -- 按 Cmd+G 打开小浮动文本字段（类似 Xcode 的 "跳转到行"），用户输入时间戳（如 "12:30" 或 "1h05m"）即可滚动转录并将音频定位到该位置。解析灵活格式："5:00"、"5m"、"300"（秒）、"1:05:30"。Enter 或 Escape 关闭。适用于在外部笔记中引用时间戳的用户（"查看23分钟处的评论"）。实现为 `.sheet` 配合 `.presentationDetent(.height(60))`。（工作量：小）

### 上下文智能

- [ ] **基于内容的自适应详情布局** -- 详情视图根据会话现有内容智能重组。有转录 + 摘要：加权分割（转录左，摘要侧栏右）。仅有转录（尚无摘要）：全宽转录配宽裕边距。有摘要但转录已删除：以摘要为中心的布局配元数据卡片。正在录音：带自动滚动转录的实时视图。使用 `ViewThatFits` 或条件布局切换配合 `matchedGeometryEffect` 实现状态间平滑过渡。应用永远不显示空面板或尴尬的占位符——每个像素服务于存在的内容。（工作量：小）

- [ ] **智能粘贴内容检测** -- 粘贴到会话标题字段或摘要编辑时检测内容类型并适配：粘贴 URL 自动格式化为链接标签，粘贴多行文本到标题时截断到第一行并显示警告 toast，粘贴时间戳字符串（"14:30"）到搜索字段时转换为时间跳转。使用 `NSPasteboard.general.string(forType:)` 在 `onChange(of:)` 中进行内容检查。防止 "粘贴错了" 的小智能。（工作量：极小）

- [ ] **计划会话的录音时长预测** -- 对从有已知时长的计划录音开始的会话，在录音按钮周围显示微妙的进度弧，随时间推进向预期结束填充。在时长的 80% 时，弧线从强调色变为琥珀色。100% 时轻柔脉动作为提醒而不打断。实现为 `Circle().trim(from:to:)` 覆盖在录音按钮上，由 `elapsedTime / expectedDuration` 驱动。用户无需看时钟就知道 "还剩多少时间"。（工作量：小）

### 播放与音频导航

- [ ] **变速播放与音调修正** -- 在 `PlaybackControlView` 中添加播放速度选择器（0.5x、0.75x、1x、1.25x、1.5x、2x），作为紧凑的分段控件。使用 `AVAudioPlayer.rate` 配合 `enableRate = true` 和 `AVAudioTimePitchAlgorithm.timeDomain` 实现自然音调修正（无花栗鼠效果）。非 1x 时在播放按钮上显示小速度标记。长按速度按钮重置为 1x。播客应用已让变速播放成为常态——没有它的转录应用感觉不完整。快捷键：`[` 和 `]` 减/增速度。（工作量：小）

- [ ] **播放时跳过静音** -- 在播放控制中添加 "跳过静音" 开关，使用现有 VAD 数据（或对音频文件的事后 RMS 分析）在播放期间自动快进超过 2 秒的静音间隔。在播放进度条上以灰色带显示跳过的区域。45 分钟有 8 分钟静音的录音变成 37 分钟。VAD 基础设施（`SimpleVAD`）已存在于实时录音——复用其阈值用于离线播放分析。Overcast 为播客普及了这个；对会议录音同样变革性。（工作量：中等）

### 无障碍即美学

- [ ] **减少动画的优雅降级** -- 审计应用中每个动画并提供 `@Environment(\.accessibilityReduceMotion)` 替代方案：弹簧按钮反馈变为即时透明度变化，转录片段滑入变为简单淡入，录音脉冲环变为静态光晕，音频电平条弹簧物理变为线性插值。在每个动画站点用 `withAnimation(reduceMotion ? .none : .spring(...))` 包裹。禁用动画后应用看起来同样精致——不是被剥离，而是更平静。这种无障碍也吸引偏好少动画的用户。（工作量：小）

- [ ] **高对比度自适应颜色 token** -- 扩展 `DS.Colors` 加入 `.accessibilityHighContrast` 变体，当 `@Environment(\.accessibilityContrast) == .high` 时激活：时间戳获得更深边框，摘要卡片强调色变得更粗，音频电平渐变转为高对比度蓝到红，会话时长点使用填充形状而非着色圆圈。定义为使用 `@Environment` 的条件 `Color` 属性。低视力用户获得*设计过的*高对比度主题，而非系统自动生成的。（工作量：小）

- [ ] **可键盘导航的转录与焦点环** -- 让每个转录片段可通过 Tab/Shift-Tab 单独聚焦，配合可见的 `.focusEffect()` 环（macOS Tahoe 的新焦点样式）。上/下箭头在片段间移动。Enter 聚焦的片段跳转音频到该时间戳。空格播放/暂停。Cmd+C 复制聚焦片段的文本。转录变得完全无需鼠标即可导航——对 VoiceOver 用户必需，对键盘优先的高级用户也很受欢迎。使用 `.focusable()` 配合 `@FocusState` 追踪活跃片段 ID。（工作量：小）

### macOS 原生交互模式

- [ ] **可自定义工具栏** -- 采用 macOS Tahoe 的可自定义工具栏 API：定义会话详情工具栏项（播放/暂停、导出、复制转录、生成摘要、分享），带稳定 `id` 值和 `customizationBehavior(.default)`。用户可右键工具栏重新排列、移除或添加项——大多数 SwiftUI 应用跳过的标准 macOS 行为。添加 `.toolbar(id: "sessionDetail")` 配合默认和可选项组。Liquid Glass 工具栏样式在 Tahoe 上免费。不需要 "分享" 按钮的用户可隐藏；频繁导出的用户可将 "导出" 移到前面。设置后零维护成本。（工作量：小）

- [ ] **Services 菜单集成转录文本** -- 将应用的转录文本视图注册为 Services 菜单提供者。用户选择转录文本时，macOS Services 菜单（右键 > Services）提供系统级操作："用选中内容新建邮件"、"添加到备忘录"、"查找"、"在 TextEdit 中打开"。大多数 SwiftUI 应用完全忽略 Services，但它是 macOS 原生的分享表单等价物——高级用户期望它。需要设置 `.focusedValue(\.selectedText, selectedTranscriptText)` 和 `FocusedValueKey`。（工作量：小）

- [ ] **会话行的丰富右键菜单** -- 丰富会话行的右键上下文菜单，不仅是删除："复制摘要到剪贴板"、"导出为 Markdown"、"以此标题新建录音"、"在 Finder 中显示音频文件"、"复制分享链接"（通过 `NSUserActivity` 的深度链接）。用 `Section` 分隔符分组相关操作，用 SF Symbol 图标修饰。丰富的上下文菜单表明应用尊重 macOS 交互模型——右键是高级用户的主要工具。（工作量：小）

### 信息密度与一目了然

- [ ] **会话行内联迷你波形** -- 在每个会话行添加 48x16pt 内联迷你波形，显示录音的音频能量轮廓。渲染存储的音频电平采样（或从音频文件惰性计算）为 `Canvas` 迷你图表。一眼区分热烈讨论（锯齿峰值）、安静讲座（平线）或有长静音的录音（间隔）。无标签、无轴线——纯视觉纹理。会话完成时计算一次并缓存为 `[Float]` 数组。（工作量：中等）

- [ ] **录音时的实时字数与说话速度** -- 在实时录音期间计时器下方显示微妙的状态行："342 字 — ~128 wpm"。从 `transcriptSegments.count` 词分割和已用时间计算。每 10 秒更新以避免分散注意力。演讲者和讲师使用每分钟字数作为节奏指南（120-150 wpm 是对话式，>180 是太快）。将录音视图变成说话教练，无需 AI——只是应用已有数据的算术。（工作量：极小）

## 第七轮 — 2026-03-24（新视角）

> 探索未覆盖的领域：写作辅助、记忆与回忆、习惯、数据可移植性、隐私 UX、多模态内容和轻量协作。每个创意瞄准第 1-6 轮未涵盖的新角度。

### 写作与编辑辅助

- [ ] **Apple Writing Tools 集成摘要** -- 在所有摘要 `TextEditor` 视图上启用 Apple Intelligence Writing Tools，设置 `.writingToolsBehavior(.complete)` 配合 `.writingToolsAllowedInputOptions([.plainText, .richText])`。用户免费获得系统级的校对、改写（友好/专业/简洁）和摘要功能——无需 LLM 调用。当前视图可能使用默认行为（`.limited`）；明确选择加入解锁完整的内联改写体验并带跟踪变更。零成本的 AI 打磨，搭载已在设备上运行的 Apple Intelligence。WritingTools 自 macOS 15.1 起可用，但大多数第三方应用未明确采用 complete 模式。（工作量：极小）

- [ ] **"优化摘要" 内联操作** -- 在每个 `SummaryCardView` 下方添加紧凑操作栏，提供三个一键文本转换（由配置的 LLM 驱动）："缩短"（压缩到 50% 长度）、"正式化"（改写为专业/第三人称口吻）、"简化"（减少术语，使用通俗语言）。每个操作将现有摘要通过 `SummarizerService` 配合有针对性的 system prompt 发送回去。结果内联替换摘要，带撤销选项（存储为 `editedContent`）。不同于引导式重新生成（需要用户输入指令）——这些是有主见的、零思考的精炼。灵感来自 Notion AI 的 "改善写作" / "缩短" / "改变语气" 操作。（工作量：小）

- [ ] **转录转散文** -- 在 `SessionDetailView` 中添加 "转换为散文" 操作，将原始带时间戳的转录通过 LLM 发送，提示改写为干净可读的叙述文档——去除填充词、错误开头、重复和时间戳痕迹，同时保留所有实质内容。输出存储为新的 `SummaryBlock`，`style: .narrative` 且 `isOverall: true`。将粗糙的口语转录变成适合博客、报告或文档的精致书面文字。不同于摘要（压缩内容）——这保留完整内容但改善可读性。（工作量：小）

### 记忆与回忆

- [ ] **"历史上的今天" 浮现** -- 应用启动时检查是否有正好一年前（或一个月前、一周前）的录音会话。如果找到，在 `SessionListView` 顶部显示微妙的可关闭横幅："1年前：产品战略会议"，附链接到该会话。使用简单的 `Calendar.current.date(byAdding: .year, value: -1, to: Date())` 查询 SwiftData。重新发现被遗忘对话的偶然性创造对应用的情感依赖——与 Apple Photos 的 "历史上的今天" 和 Facebook Memories 相同的心理学。无需 AI，只是日期查询。（工作量：极小）

- [ ] **会话摘要通知** -- 生成每周摘要，总结录音活动：会话数量、总时长、热门话题（从标签或 LLM 分析标题提取）和本周会话中的待办事项。通过 `UNUserNotificationCenter` 每周一早上发送本地通知，带深度链接到摘要视图。对每天录音的用户，这种每周反思创造 "闭合循环" 的仪式。灵感来自 Toggl 的每周邮件报告和 Daylio 的每周心情总结。（工作量：中等）

- [ ] **自然语言模糊回忆搜索** -- 用自然语言理解增强现有 `.searchable()`：输入 "上个月讨论预算的那次会议" 使用 LLM 提取意图（主题："预算"，时间范围："上个月"）并用适当的日期筛选 + 全文匹配查询 SwiftData。LLM 不可用时回退到标准子字符串搜索。不同于 "跨会话知识搜索"（第一轮，综合跨会话答案）——这是关于*找到*正确的会话，而非回答关于它的问题。（工作量：中等）

### 习惯与参与

- [ ] **录音连续记录追踪** -- 追踪连续有至少一个录音会话的天数，在侧栏页脚或菜单栏显示连续天数："7天连续" 配合小火焰图标（SF Symbol `flame.fill`）。达到里程碑（7、30、100天）时显示温和的庆祝动画（纸屑或光晕脉冲）。持久化连续数据于 `@AppStorage`（仅 `lastRecordingDate` 和 `currentStreak` 整数）。研究显示 44% 的用户在中断连续后流失，所以包含 "连续冻结" 宽限期（1天缺失不中断）。Duolingo 部分靠连续心理学建立了 120亿美元的生意——轻量版本几乎不花成本。（工作量：小）

- [ ] **每日语音日记提示** -- 在用户配置的时间（默认：早上 8 点），显示本地通知带反思提示："今天你在想什么？"、"你感恩什么？"、"你今天的首要任务是什么？" 点击通知打开 Notetaker 并以提示为会话标题开始录音。循环 30 个精选提示。在 `@AppStorage` 中存储提示历史以避免月内重复。将应用从会议工具变成每日个人练习。SparkPod 和 Reflectly 等音频日记应用证明这推动日活。通过设置中的 "每日日记" 开关切换。（工作量：小）

- [ ] **会话里程碑庆祝** -- 用简短优雅的致意庆祝录音里程碑："你已录制 10 个会话！"（首个里程碑）、"已捕获 50 小时音频"、"100 个会话——你是超级用户"。在 `SessionListView` 中使用 `.overlay()` 配合温暖渐变背景和 SF Symbol（`trophy.fill`、`star.fill`）显示为一次性可关闭卡片。里程碑存储为 `@AppStorage` 中的 `Set<String>`，每个只显示一次。通过低频和真诚而非强推避免游戏化疲劳。（工作量：极小）

### 数据可移植性

- [ ] **完整备份与恢复归档** -- 在设置中添加 "导出全部数据"，创建单个 `.notetaker` ZIP 归档，包含：所有音频文件（M4A）、所有会话的 JSON 清单（含转录、摘要、标签和待办事项）加应用偏好设置。"导入归档" 读取 ZIP 并恢复到 SwiftData，按会话 UUID 跳过重复。使用 `FileManager` + `ZIPFoundation`（或 shell `zip`/`unzip`）。这是用户迁移机器、恢复数据丢失或只想安心的核武器选项。不同于按会话 Markdown 导出（第一轮）——这是整个应用状态的完整可逆备份。（工作量：中等）

- [ ] **导入音频文件进行转录** -- 在文件菜单添加 "导入音频"（`Cmd+I`），接受 `.m4a`、`.mp3`、`.wav`、`.aiff` 文件。创建新的 `RecordingSession` 并导入文件，然后像刚完成的实时录音一样运行 ASR + 摘要。将文件复制到应用的录音目录。将 Notetaker 从 "仅限实时录音" 变成通用音频转录工具。不同于 "Finder 快速操作"（第五轮，Finder 扩展）——这是应用内等价物，更容易实现且可通过菜单栏发现。（工作量：小）

- [ ] **多格式转录复制** -- 扩展现有转录复制操作以通过子菜单或修饰键变体提供格式选择：纯文本（当前）、带时间戳的 Markdown（`**[03:42]** Speaker: text`）、SRT 字幕格式（给视频编辑器）和 CSV（给电子表格分析）。每种格式是现有 `TranscriptSegment` 数据通过 `TranscriptExporter` 的简单字符串转换。SRT 和 CSV 导出以最少代码解锁全新用例（字幕制作、数据分析）。（工作量：小）

### 隐私与安全 UX

- [ ] **数据仪表盘与存储分析** -- 在设置中添加 "你的数据" 区域，精确显示应用存储了什么数据和在哪里：总音频文件大小、会话数量、转录字数、摘要数量和磁盘使用分析（音频 vs 数据库 vs 崩溃日志），附条形图。本地配置时显示 "数据从不离开这台 Mac" 标记配锁图标；配置外部 LLM 时显示 "数据发送到 [provider URL]" 配警告图标。灵感来自隐私优先的 UX 模式——让数据流可见且可审计。*看到*数据是本地的用户才*信任*它是本地的。（工作量：小）

- [ ] **逐会话隐私标签** -- 在每个会话行显示小隐私标记，指示数据流向：锁图标（`lock.fill`）表示使用设备端 Foundation Models 或本地 Ollama 摘要的会话，云图标（`cloud.fill`）表示通过 OpenAI/Anthropic API 处理的会话。在 `SummaryBlock` 中存储每个摘要使用的 LLM 提供者为 `providerLabel: String?` 字段。用户可筛选 "仅本地" 会话。让每个会话的隐私影响一目了然，而非埋在设置中。（工作量：小）

- [ ] **临时录音模式** -- 在录音控制栏添加 "消失的会话" 开关（幽灵图标，`eye.slash.fill`）。启用后，会话在可配置的保留期（1 小时、24 小时、7 天）后自动删除——音频文件、转录和摘要全部移除。使用 `RecordingSession` 上的 `retainUntil: Date?` 字段和应用启动时的清理扫描。适用于敏感对话（HR、法律、心理咨询），用户想临时访问转录但不想永久存储。不同于会话加密（第一轮，保护静态数据）——这完全移除数据。（工作量：小）

### 多模态内容

- [ ] **截图附件绑定会话** -- 允许用户将截图粘贴或拖动图片到 `SessionDetailView`，以时间戳附加到活跃会话。在 `RecordingSession` 上的新 `attachments: [String]` 数组中存储图片文件路径。在转录下方以缩略图带显示，点击放大。实时录音期间，按全局热键（Cmd+Shift+S）通过 `CGWindowListCreateImage` 捕获截图并自动附加。适用于录制演示文稿走查、设计评审或软件演示——视觉上下文与口语同样重要。（工作量：中等）

- [ ] **URL 链接捕获与预览** -- 添加 "添加链接" 按钮（或检测粘贴到会话备注字段的 URL），存储与会话关联的网页 URL。通过 `LPMetadataProvider`（LinkPresentation 框架）获取和缓存页面标题和 favicon 用于富预览显示。会议录音期间，用户不断引用文档、PR、工单和 wiki 页面——将这些链接与转录一起捕获创造完整的会话制品。在 `SessionDetailView` 中显示为紧凑链接列表，点击在默认浏览器中打开。（工作量：小）

### 轻量分享

- [ ] **通过 macOS 分享表单分享摘要** -- 在 `SummaryCardView` 上添加分享按钮，调用 `NSSharingServicePicker` 以摘要文本为载荷。用户可两次点击发送到信息、邮件、备忘录、AirDrop 或任何已安装的分享扩展。当前唯一分享方式是手动复制粘贴。原生分享表单自动处理格式适配（信息用纯文本，邮件用富文本）。不同于 "分享为精美摘要卡片"（第一轮，生成自定义 HTML 卡片）——这使用系统分享基础设施实现零摩擦分享。（工作量：极小）

- [ ] **导出会话为可分享 HTML 页面** -- 从会话生成自包含的单文件 HTML 页面（标题、日期、带时间戳的格式化转录、摘要、待办事项），使用嵌入的 CSS 样式表。文件可在任何浏览器中打开、作为邮件附件发送或托管在简单 Web 服务器上。无 JavaScript 依赖，无构建步骤——只是将字符串插入 HTML 模板。不同于 Markdown 导出（给 PKM 工具）——这是给只需在浏览器中阅读的非技术收件人的。（工作量：小）

- [ ] **AirDrop 快速分享会话** -- 将 `RecordingSession` 注册为 `Transferable` 类型，导出为 `.notetaker` 包（包含音频 + 转录 JSON + 摘要文本的 ZIP）。用户 AirDrop 会话到另一台运行 Notetaker 的 Mac 时，接收应用将其导入为新会话。适用于想要无任何云服务分享会议笔记的纯 Mac 团队。使用 `Transferable` 协议配合 `ProxyRepresentation` 用于系统分享表单和 AirDrop。（工作量：中等）
